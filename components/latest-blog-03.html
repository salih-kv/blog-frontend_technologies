Is it hard to learn Angular with Javascript from zero?

But when you encounter a legacy app, or a corporate app, with Angular 10, you realize that you need to re-learn your knowledge with the older version. What do you need to re-learn, and why it is important to know some basics of AngularJS and the history of Javascript?

How you should start your learning Path?

I would always start with understanding Javascript. It is great to start with a wrap-up course and learn the Javascript es6+. Then you can eventually skip some details and come to them later as it can be more beneficial to get back to the details once you start working on some project because then the light in your head will pop up and you will start to chain some pieces of knowledge together.

Secondly, I recommend playing with fetch API and showing data, then altering it. The really best way is to start with something dummy, there are a lot of resources such as this: https://random-data-api.com/

Don’t jump directly into Angular

After you get your basics in Javascript, don't jump directly to Angular, but try to take 2–4 weeks of learning Jquery and play with some SVG libraries, just to get the feeling of how you include the libraries into in your index.html, how does it work. Try to check what files are calling, how big the libraries are etc.

It is also possible that at that point you will find some specific topic in Javascript you don’t understand, then I recommend checking this topic and reviewing your Javascript knowledge again

Now you are ready to start learning Angular

When your foundation is strong, you can jump to learn Angular or any other front-end framework. The best thing to do now is to go via the official Angular tutorial. In that tutorial, you can see a small project developed with Angular. Then when you finish the project, it's great to start reading the documentation and think about the concepts: Angular components, Directives, and Pipes. Or you can check some courses on Udemy.

I’ve created a theoretical concept to handle in Angular by category:

Beginners concepts: Angular — Components, Directives, RxJs, pipe vs template functions

Advanced concepts: Project structure, Lazy loading, Subjects and Data streams, NgRx, unit tests, nx monorepo, Standalone components

Expert concepts: Decide whether to use a state management library or have internal data in an observable, decide on where the data logic will be stored (if in components, service facades, or helper methods), know your logic layers, decide about typescript configuration, angular configuration. Decide about your zone.js and make speed optimizations….

When you feel that you understand all the concepts, you can check AngularJS and see how it was implemented the old-fashioned way. By understanding this you might even gain a better understanding of the current decision and the direction of the front-end in the future.

And now the weird part again…

It is possible that once you master your framework, you might want to go even deeper into Javascript. So now is the time to re-learn some javascript concepts and forget about Ajax… Start checking the fetch() API, the promise vs. callback vs async/await. Also, you can try to implement the concept of observable and subscription in plain javascript to gain a better understanding of the concept. And definitely, I recommend checking why IIFEE and closures were so popular in Javascript.

Still not done?

Well, you might think that you become a Javascript expert by now. But wait. Oh shit. We’ve forgotten to mention the Canvas and SVG animations and… game development in Javascript.

In the end, I want to emphasize the fact that Javascript is like a never-ending story. There are only a few people who can master it all and knows all the corners to the detail. Javascript is such a general language today, that you need to take your niche first and become an expert in that niche and only then you can start to deepen your knowledge. Otherwise, you can be lost in time and even be frustrated.
